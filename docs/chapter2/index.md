# 第 2 章：如何学习和评价一个开源框架

第一章建立了 Android 开源框架的全景地图。本章进一步回答一个更实用的问题：面对一个陌生框架，应该如何学习、如何读源码、如何判断它是否适合当前项目？

本章分为两节：

| 小节 | 主题 | 内容 |
| --- | --- | --- |
| [第 2.1 节：如何学习一个开源框架](./chapter2_1.md) | 学习方法 | 从问题、核心抽象、工程接入、调用链、扩展点、测试和 Issue 中抓住框架主线 |
| [第 2.2 节：框架评价与选型维度](./chapter2_2.md) | 选型方法 | 从场景、功能、工程、团队、迁移、封装和治理维度判断框架是否适合项目 |

## 本章学习目标

学习完本章后，你应该能够：

- 用统一的问题清单快速理解一个开源框架；
- 从 README、Sample、测试和源码入口中抓住框架主线；
- 画出框架的最小调用链，避免无差别通读源码；
- 使用评价矩阵比较同类框架；
- 判断框架是否适合当前项目、团队和长期维护目标；
- 为后续图片、网络、异步、依赖注入等章节建立统一分析方法。

## 推荐阅读方式

建议先阅读 [第 2.1 节](./chapter2_1.md)，建立学习框架的方法；再阅读 [第 2.2 节](./chapter2_2.md)，理解真实项目中的框架评价和选型。

后续每个具体框架章节，都会沿用本章的方法：

```text
问题背景
  -> 核心抽象
  -> 最小使用
  -> 源码主线
  -> 同类对比
  -> 工程选型
```
